# Phase 03：界面稳定性与交互一致性

## 1. 目标（Goal）
修复不同子页面切换造成的主容器抖动与高度跳变问题，提升连续操作体验。

## 2. 对应 Skill（Skill Mapping）
- `frontend-skill`：布局稳定性与跨模式交互一致性优化。

## 3. 实施过程（Steps）
1. 识别“生成 / 美化 / 解析”三模式内容高度差异。
2. 为模式内容容器设置统一最小高度基线（`min-h-[580px]`）。
3. 保持原有视觉风格，不破坏既有组件布局关系。

## 4. 变更清单（Changed Files）
- `QCodeMerged.html`

## 5. 验证与结果（Validation）
- 三模式来回切换时主面板尺寸稳定。
- 页面重排抖动明显减少。
- 不影响既有批处理及下载功能。

## 6. 后续建议（Next）
- 可进一步基于断点（移动端/桌面端）制定分级最小高度策略。
