# Phase 02：二维码美化批处理能力

## 1. 目标（Goal）
在既有统一页面基础上，新增“二维码美化”模式，并支持多文件批处理的上传、解析、生成与下载闭环。

## 2. 对应 Skill（Skill Mapping）
- `frontend-skill`：批处理界面信息层级、交互密度与视觉一致性。
- `code-change-log`：功能说明与测试路径补充。

## 3. 实施过程（Steps）
1. 在顶部模式切换中加入“二维码美化”。
2. 扩展拖拽区，支持多文件一次导入。
3. 建立逐项解析 -> 逐项渲染的处理流水。
4. 增加单项下载（SVG / PNG）与批量下载（PNG）。
5. 同步多语言文案，保证中英双语一致。

## 4. 变更清单（Changed Files）
- `QCodeMerged.html`
- `task.md`
- `walkthrough.md`

## 5. 验证与结果（Validation）
- 多文件拖入后可连续处理，不阻塞主界面。
- 每个结果项支持独立下载。
- 批量下载可覆盖当前处理列表。
- 多语言切换后批处理说明文案一致。

## 6. 后续建议（Next）
- 增加异常文件类型提示与失败项重试入口。
